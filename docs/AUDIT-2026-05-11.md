## Systemic Issues

### 1. Locale fallback gap (~40+ instances across 25+ files)
`obj.name[locale]` without `|| obj.name.en` fallback. The `tl()` helper exists and handles this correctly but is not used in these spots. 7 of 10 active locales see "undefined" rendered.

### 2. UTC-parse + local-accessor pattern (8 instances)
`new Date("YYYY-MM-DD")` creates UTC midnight, then local-TZ methods (`getDate()`, `setDate()`, `getMonth()`) are called. Off-by-one errors on UTC- servers. Fix: use `getUTCDate()`/`setUTCDate()` or parse manually.

### 3. getMasa() solar approximation in kundali (2 instances)
Lesson M was violated again — the same bug documented in CLAUDE.md from the April audit. Two separate files use the solar approximation for masa display.

### 4. CRON_SECRET inconsistency (4 of 11 routes)
Two auth patterns coexist: `if (cronSecret && authHeader !== ...)` (skip if unset) vs `if (authHeader !== ...)` (reject if unset). The first pattern is insecure.

### 5. Timezone function misuse (1 remaining instance)
`sign-shift/page.tsx` uses `resolveCurrentLocationTimezone` for birth computation — should be `resolveBirthTimezone`. Same bug class as the Diksha Acharya regression.

### 6. AI API cost exposure (1 instance)
`api/horoscope/route.ts` calls Claude API with no auth and no rate limiting. Any visitor can trigger billable AI calls.
