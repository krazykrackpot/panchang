# Transit Article Pages — Design Spec

**Date:** 2026-04-29
**Status:** Approved

## Overview

Create editorial-style transit articles published ahead of major planetary transits. Each article covers the transit's general significance + detailed effects on all 12 Moon signs. Content is AI-generated at dev time (not runtime), stored in a TypeScript data file, and rendered via a dynamic route template.

**First article:** Jupiter → Cancer, June 2, 2026.

## Route

`/[locale]/learn/transits/[slug]` — dynamic route under existing `/learn/transits/` parent.

Examples:
- `/en/learn/transits/jupiter-in-cancer-2026`
- `/hi/learn/transits/jupiter-in-cancer-2026`

## Data Source

**File:** `src/lib/content/transit-articles.ts`

A map of slug → article data. All commentary is pre-written and committed — no runtime API calls.

```ts
interface MoonSignEffect {
  rashiId: number;           // 1-12
  house: number;             // which house the transiting planet occupies from this Moon sign
  headline: string;          // e.g., "Career Breakthrough"
  body: string;              // 2-3 paragraphs, editorial tone
  dosAndDonts: string[];     // 3-4 actionable bullet points
  remedy: string;            // one-line mantra or remedy
}

interface KeyDate {
  date: string;              // ISO date
  event: string;             // e.g., "Jupiter turns retrograde"
  significance: string;      // 1-2 sentence explanation
}

interface TransitArticle {
  slug: string;
  planetId: number;
  fromSignId: number;
  toSignId: number;
  year: number;
  exactDate: string;
  endDate: string;
  duration: string;          // human-readable, e.g., "~5 months (Jun 2 – Nov 1, 2026)"
  title: { en: string; hi: string };
  metaDescription: { en: string; hi: string };
  overview: { en: string; hi: string };       // 3-4 paragraphs
  generalThemes: { en: string; hi: string }[];  // 4-6 bullet points
  moonSignEffects: MoonSignEffect[];            // 12 entries (English only — Hindi content uses tl() fallback)
  keyDates: KeyDate[];
  retrogradeNote?: string;
  publishDate: string;       // ISO date for Article JSON-LD
}

export const TRANSIT_ARTICLES: Record<string, TransitArticle> = { ... };
```

   - Planet icon (GrahaIconById, 64px) + planet color glow

3. **General Themes**

6. **Retrograde Note** (conditional)

7. **Cross-links**
   - "View Transit Timeline →" link to `/transits`
   - "Generate Your Kundali →" link to `/kundali`
   - "Other Transit Articles" links to other published articles

## Content Generation

The editorial content for each article is generated by prompting Claude at dev time (not via API at runtime). The content is then committed to `transit-articles.ts` as string literals.

**Content style guidelines:**
- Write as a knowledgeable astrologer speaking to an interested reader
- Use natural, flowing prose — not bullet-point lists for the main body
- Reference classical texts casually (e.g., "As Parashara notes..." or "The Brihat Jataka suggests...")
- Include practical advice, not just predictions
- Each Moon-sign section should feel specific and actionable, not generic
- Vary sentence structure and paragraph length — avoid the AI "list of three" pattern
- Use concrete examples: "If you're in a corporate job, expect..." rather than "Career changes may occur"

## Not In Scope

- Hindi translations of Moon-sign effect bodies (English only for v1 — Hindi readers get the overview in Hindi, effects in English)
- Runtime content generation via API
- User comments or reactions
- Transit notification system
