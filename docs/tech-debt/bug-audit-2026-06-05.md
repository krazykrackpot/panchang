## Genuine findings (10)

### MEDIUM — silent failures + TOCTOU (4)

#### B1. `matching/Client.tsx` kundali Promise.all swallows error

- **File**: `src/app/[locale]/matching/Client.tsx:230-240`
- **Symptom**: When both partner kundalis fail to generate, error is logged
  but no state update fires. UI shows a stuck loading skeleton or blank
  page, user can't tell anything went wrong.
- **Fix**: Add an `error` state, set on `.catch()`, render an "unable to
  load chart" message with retry button.

#### B2. `cron/vrat-reminder` TOCTOU on rollback

- **File**: `src/app/api/cron/vrat-reminder/route.ts:245-372`
- **Symptom**: Reads `pref.last_day_before_reminder_date`, awaits
  `sendDayBefore()`, on failure rolls back to the stale snapshot value.
  If a concurrent cron tick updated the column during the await, the
  rollback writes back an earlier value.
- **Fix**: Either refetch the pref row before rollback OR drop the
  rollback entirely (idempotency through the unique `(user_id, vrat_id,
  occurrence_date, kind)` index on `user_notifications`).

#### B3. `tippanni-llm` comparison-mode Promise.all loses both on either failure

- **File**: `src/app/api/tippanni-llm/route.ts` around line 271
- **Symptom**: Comparison mode fans out Sonnet + Opus syntheses via
  `Promise.all`. If either model fails (rate limit, timeout), the
  other's result is discarded.
- **Fix**: `Promise.allSettled` + return whichever succeeded, with a
  note for the missing one.

#### B4. `cron/email-alerts` dedup orphan on cron crash

- **File**: `src/app/api/cron/email-alerts/route.ts:112-141`
- **Symptom**: Pattern: insert dedup row → send email → delete row on
  failure. If the cron handler crashes (Vercel 5-min timeout, OOM, OOB
  error) between insert and send, the row persists and no future cron
  tick will send because the dedup blocks it.
- **Fix**: Either (a) move dedup to a flag on the pref row that
  `sendEmail()` claims-then-commits, OR (b) accept the rare orphan and
  add a sweeper cron that deletes orphan dedup rows older than 24h with
  no corresponding email-log entry.

### LOW — defensive hardening (6)

#### B5. `dasha-prognosis` punctuation duplicates on no-split

- **File**: `src/lib/tippanni/dasha-prognosis.ts:876-877`
- **Code**:
  ```ts
  const enFirst = mahaNature.en.split('. ')[0] + '.';
  const hiFirst = mahaNature.hi!.split('। ')[0] + '।';
  ```
- **Symptom**: If `en` has no sentence break, `split` returns the whole
  string and we append a duplicate `.`. Cosmetic only.
- **Fix**: Append only if the original didn't already end with the
  punctuation.

#### B6. `notification-engine.daysUntil` NaN on invalid date string

- **File**: `src/lib/personalization/notification-engine.ts:40, 47, 52, 91`
- **Symptom**: `new Date(maybeUndefinedString).getTime()` returns NaN
  if dateStr is malformed. Math.ceil(NaN/...) is NaN. Comparisons silently
  pass through.
- **Triage note**: Current callers all guard against missing endDate.
  But the function signature accepts `string` with no validation.
- **Fix**: ISO regex pre-check at the top of `daysUntil` / `daysSince`,
  return 0 (or `Number.NaN` with a console.warn) on malformed input.

#### B7. `kundali-calc` planet-dignity arrays fragile to extension

- **File**: `src/lib/ephem/kundali-calc.ts:362, 367`
- **Code**:
  ```ts
  const digBalaHouse = [10, 4, 10, 1, 1, 4, 7][p.planet.id];
  const naisargikaBala = [60, 51, 17, 25, 34, 42, 8][p.planet.id] || 30;
  ```
- **Symptom**: Arrays are 7 entries (Sun-Saturn, no Rahu/Ketu). If
  someone extends shadbala to include nodes, indices 7-8 silently
  return undefined. The `|| 30` fallback masks it for naisargikaBala
  but digBalaHouse goes silently wrong.
- **Fix**: Move both into the canonical `dignities.ts` as a typed
  `Record<PlanetId, number>` and use it via lookup. Out-of-range
  becomes a TS error.

#### B8. `convergence-llm/synthesizer.ts` JSON.parse on streaming chunk

- **File**: `src/lib/tippanni/convergence-llm/synthesizer.ts:142`
- **Symptom**: Streaming chunk parsed without try/catch. Anthropic
  occasionally sends partial chunks or keep-alive frames; `JSON.parse`
  throws and the stream loop aborts mid-synthesis.
- **Fix**: Wrap in try/catch, return early on parse failure (chunk will
  be retried in the next event).

#### B9. `life-areas-enhanced` house-lord nullable chain

- **File**: `src/lib/tippanni/life-areas-enhanced.ts:164-167, 267-268, 313-316`
- **Code**:
  ```ts
  const lord2Name = houses.find(h => h.house === 2)?.lordName.en ?? '';
  ```
- **Symptom**: If `lordName` is undefined (rare, but theoretically
  possible if house data is partial), the `?.lordName.en` access
  throws. Optional chain stops at `?.lordName`, then `.en` deref on
  undefined.
- **Fix**: `?.lordName?.en ?? ''`.

#### B10. `personal-panchang` taraBala NaN

- **File**: `src/lib/personalization/personal-panchang.ts:161`
- **Code**: `const taraNumber = (diff % 9) + 1;`
- **Symptom**: If `diff` is NaN from an upstream miscompute, `NaN % 9`
  is NaN, propagates through tara scoring. Upstream has guards but
  the function doesn't.
- **Fix**: `if (!Number.isFinite(diff)) return null` before the modulo.

## False positives (6)

Documented so future audits don't re-flag them.

### FP1. Async #1 — welcome-email fire-and-forget

`src/app/api/user/profile/route.ts:382` — the `.catch()` handler that
logs without rethrowing is INTENTIONAL best-effort design (claim is
already set; email failure shouldn't block the API response). Comment
at the call site documents this.

### FP2. Async #3 — horoscope inner `.catch(() => null)`

`src/app/[locale]/horoscope/HubClient.tsx:348-354` — the inner
`.catch(() => null)` swallows per-rashi fetch failures and the
component renders nulls gracefully. This is intentional; the outer
fan-out shouldn't fail just because one rashi's horoscope is missing.

### FP3. Async #7 — horoscope chunked Promise.all

`src/app/api/horoscope/route.ts:105-136` — agent's own re-read noted
the loop correctly awaits each chunk. No bug.

### FP4. Async #8 — family-synthesis in-flight dedup

`src/app/api/family-synthesis/route.ts:41-46` — agent claimed a race
between `get()` and `set()`. In JS's single-threaded event loop, the
sync portion between `const promise = (async () => {...})()` and
`inflightSynthesis.set(user.id, promise)` cannot be interleaved by
another request handler. The race is impossible.

### FP5. Boundary #1 — MONTH_ABBR out-of-bounds

`src/lib/ephem/panchang-calc.ts:1031, 1073` — `jdToLocalDate` uses
the Meeus algorithm which is mathematically guaranteed to produce
month ∈ [1, 12]. Index 12 in MONTH_ABBR[mo-1] is unreachable for
any valid JD input.

### FP6. Boundary #5 — nakshatra-28 floating-point edge

`src/lib/kundali/chart-narrative.ts:198` — agent claimed `floor(360/13.33)`
could return 27 from floating-point error. In practice, JS `%` is
well-defined and `360 % 360 === 0` exactly; the longitude can't be
`360 - epsilon` because the input is already mod-360. Hypothetical edge
doesn't trigger.
